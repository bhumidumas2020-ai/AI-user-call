  SECTION 1 — WHAT THIS PROJECT DOES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

HiveStaff AI Voice Agent is an intelligent, emotion-aware phone support system
for HiveStaff, a workforce management platform.

  A person calls a Twilio phone number
  → The AI answers, listens, understands the question
  → Searches your HiveStaff PDF documentation using semantic search (RAG)
  → Detects the caller's emotional tone
  → Replies with a natural-sounding human voice
  → Records the full call and logs every conversation turn

Key capabilities:
  • Inbound and outbound calls via Twilio
  • Speech-to-Text using Groq Whisper (whisper-large-v3-turbo)
  • Document-grounded answers via RAG (FAISS + SentenceTransformers)
  • LLM responses via Groq LLaMA 3.1 8B Instant — short, conversational
  • Emotion detection with 7 emotion categories
  • Natural Text-to-Speech via Groq PlayAI voices
  • Full call recordings (both AI + user sides) stored locally
  • Live call log dashboard in your browser
  • Conversation memory — 16-message rolling window per session

  SECTION 3 — CALL FLOW (STEP BY STEP)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  STEP 1  │  Caller dials your Twilio phone number
           │
  STEP 2  │  Twilio sends HTTP POST to your ngrok URL → /voice endpoint
           │
  STEP 3  │  Flask /voice handler runs:
           │    a. Starts full call recording (Twilio API)
           │    b. Plays pre-generated greeting audio
           │    c. Instructs Twilio to record the caller's speech (max 45 sec)
           │    d. Sets /process_speech as the callback when speech ends
           │
  STEP 4  │  Caller speaks. Twilio detects silence (2 sec) → stops recording
           │
  STEP 5  │  Twilio posts recording URL to /process_speech
           │
  STEP 6  │  Flask /process_speech handler runs:
           │    a. Downloads the audio clip from Twilio
           │    b. Sends audio to Groq Whisper → gets transcript text
           │    c. Runs emotion detection on the transcript (keyword rules)
           │    d. Embeds the transcript → FAISS search → retrieves top 3 chunks
           │    e. Builds system prompt with context + emotion instruction
           │    f. Calls Groq LLaMA 3.1 8B → gets AI reply (max 80 tokens)
           │    g. Sends reply text to Groq PlayAI TTS → generates .wav audio
           │    h. Saves .wav to /tmp/audio_responses/
           │    i. Returns TwiML telling Twilio to play audio + listen again
           │
  STEP 7  │  Twilio plays AI audio to caller
           │
  STEP 8  │  Loop back to STEP 4 for next turn
           │
  STEP 9  │  When call ends, Twilio posts full call recording to
           │  /recording_status → Flask downloads and saves it locally
           │
  STEP 10 │  Browser dashboard at /logs shows full transcript + audio player

  SECTION 4 — RAG PIPELINE (DOCUMENT INTELLIGENCE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  INDEXING PHASE (runs once at startup — Cell 2.5):

  hivestaff_docs.pdf
       │
       ▼
  Docling DocumentConverter
       │  Converts PDF → clean Markdown text
       │  Fallback: PyMuPDF if Docling gives empty output
       ▼
  Chunker
       │  chunk_size    = 400 characters
       │  chunk_overlap = 80 characters
       │  Creates overlapping text windows to preserve sentence continuity
       ▼
  SentenceTransformer (all-MiniLM-L6-v2)
       │  Embeds each chunk → 384-dimensional float32 vector
       │  Runs on Colab GPU (free)
       ▼
  FAISS IndexFlatIP
       │  Inner product (cosine after L2 normalisation)
       │  All chunk vectors stored in memory
       ▼
  Index ready — typical result: 200–400 chunks per 30-page PDF


  QUERY PHASE (runs on every caller turn):

  User speech transcript
       │
       ▼
  Embed with same SentenceTransformer model
       │
       ▼
  FAISS search — top 3 chunks by cosine similarity
       │  Score threshold: 0.20 minimum (chunks below this are skipped)
       │
       ▼
  Retrieved text chunks concatenated as context block
       │
       ▼
  Injected into LLaMA system prompt as === HIVESTAFF DOCUMENTATION ===
       │
       ▼
  LLaMA answers ONLY from this context
       (if no relevant chunk found → polite fallback message)

  SECTION 5 — EMOTION DETECTION & VOICE MAPPING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Detection method: keyword matching (no extra API call, zero extra cost)

  User says words like...         →  Detected as   →  AI responds as  →  Voice
  ─────────────────────────────────────────────────────────────────────────────
  "frustrated", "useless",        →  angry          →  sad             →  Aaliyah-PlayAI
  "unacceptable", "pathetic"

  "sad", "disappointed",          →  sad            →  sad             →  Aaliyah-PlayAI
  "heartbroken", "upset"

  "worried", "urgent", "bug",     →  fearful        →  calm            →  Aaliyah-PlayAI
  "not working", "stuck"

  "great", "amazing", "love",     →  happy          →  happy           →  Cole-PlayAI
  "thank you so much"

  "excited", "can't wait"         →  excited        →  excited         →  Cole-PlayAI

  "shocked", "no way", "really"   →  surprised      →  surprised       →  Cole-PlayAI

  (no keyword matched)            →  neutral        →  neutral         →  Cole-PlayAI

  Emotion instruction is appended to the LLM system prompt to shape tone.
  Voice is selected per emotion before TTS generation.

  10.2  VOLUME CALCULATIONS (daily, at 50 calls)
  ──────────────────────────────────────────────────────────────────────────

  Audio minutes per day:
    = calls × duration
    = 50 × 2
    = 100 minutes/day

  Total conversation turns per day:
    = calls × turns per call
    = 50 × 4
    = 200 turns/day

  LLM input tokens per day:
    = total turns × input tokens per turn
    = 200 × 400
    = 80,000 tokens/day

  LLM output tokens per day:
    = total turns × max output tokens per turn
    = 200 × 80
    = 16,000 tokens/day
    (Note: actual output is often less than max_tokens.
     This is the worst-case / upper-bound calculation.)

  TTS characters per day:
    = total turns × reply length in characters
    = 200 × 110
    = 22,000 characters/day

  10.3  GROQ API COSTS (Line by Line)
  ──────────────────────────────────────────────────────────────────────────

  LINE 1 — Whisper STT (Speech-to-Text)
  ───────────────────────────────────────
  Service:    Groq whisper-large-v3-turbo
  Billing:    Per audio minute processed
  Rate:       $0.001 per minute

  Daily calculation:
    = audio minutes per day × rate
    = 100 min × $0.001/min
    = $0.1000 per day

  Monthly:
    = $0.1000 × 30
    = $3.00 per month

  ─────────────────────────────────────────────────────────
  LINE 2 — LLaMA 3.1 8B Input Tokens
  ───────────────────────────────────
  Service:    Groq llama-3.1-8b-instant (input side)
  Billing:    Per input token sent to the model
  Rate:       $0.05 per 1,000,000 tokens

  Daily calculation:
    = input tokens per day × (rate per token)
    = 80,000 × ($0.05 ÷ 1,000,000)
    = 80,000 × $0.00000005
    = $0.0040 per day

  Monthly:
    = $0.0040 × 30
    = $0.12 per month

  ─────────────────────────────────────────────────────────
  LINE 3 — LLaMA 3.1 8B Output Tokens
  ─────────────────────────────────────
  Service:    Groq llama-3.1-8b-instant (output side)
  Billing:    Per output token generated by the model
  Rate:       $0.08 per 1,000,000 tokens

  Daily calculation:
    = output tokens per day × (rate per token)
    = 16,000 × ($0.08 ÷ 1,000,000)
    = 16,000 × $0.00000008
    = $0.0013 per day

  Monthly:
    = $0.0013 × 30
    = $0.04 per month

  ─────────────────────────────────────────────────────────
  LINE 4 — PlayAI TTS (Text-to-Speech)
  ──────────────────────────────────────
  Service:    Groq playai-tts (Aaliyah-PlayAI / Cole-PlayAI)
  Billing:    Per character of text converted to audio
  Rate:       $22.00 per 1,000,000 characters

  Daily calculation:
    = TTS characters per day × (rate per character)
    = 22,000 × ($22 ÷ 1,000,000)
    = 22,000 × $0.000022
    = $0.4840 per day

  Monthly:
    = $0.4840 × 30
    = $14.52 per month

  Note: TTS is the LARGEST Groq cost by far.
  Reducing reply length from 110 to 70 characters saves ~$5/month at 50 calls.

  10.8  COST OPTIMISATION TIPS
  ──────────────────────────────────────────────────────────────────────────

  1. BIGGEST SAVING — Reduce AI reply length (TTS cost)
     Current:  110 chars/reply  → $14.52/month (Groq TTS)
     If 70ch:   70 chars/reply  → $9.24/month  → saves $5.28/month
     If 50ch:   50 chars/reply  → $6.60/month  → saves $7.92/month
     Change: reduce chars_per_reply assumption and keep AI prompt to 1 sentence.

  2. BIGGEST SAVING — Disable recording when not needed
     Recording adds $7.50/month at 50 calls/day.
     If calls are for support QA only occasionally, record selectively.

  3. Reduce turns per call
     If average conversation is 2 turns (not 4):
     Halves LLaMA, TTS, and Whisper costs (saves ~$0.29/day = $8.70/month).

  4. Reduce TOP_K_CHUNKS from 3 to 2
     Each chunk is ~400 chars of context included in LLM input.
     TOP_K_CHUNKS=2 vs 3 cuts ~133 input tokens per turn
     → saves ~$0.001/day at 50 calls → minor saving but faster responses.

  5. Move to Twilio PAYG vs monthly if call volume is very low
     At under 10 calls/day the fixed phone number cost ($1.15/mo) is the
     relatively biggest fixed overhead — can't reduce below this.

  6. For production scale (500+ calls/day) — negotiate Twilio volume pricing
     Twilio offers discounted rates for high-volume customers.
     At 1,000 calls/day contact Twilio enterprise team.
